Overview

What is Prism?

Prism is an extension-first visual transformation platform. It scans the active page, reads visible DOM elements and computed styles, groups visual patterns into semantic roles, and applies a curated or custom theme without modifying the website source.

Prism is not a generic CSS editor, full website builder, accessibility certification suite, or AI redesign generator. Its job is to transform a live website's visual system while showing the accessibility impact of that transformation.

Setup

Install the Extension

  1. Open the Chrome or Edge extensions page.
  2. Enable developer mode.
  3. Choose Load unpacked.
  4. Select the Prism extension folder.
  5. Open a conventional website and click the Prism toolbar icon.

Chrome and Edge are the initial targets. Firefox and Safari are deferred until the core transformation workflow is validated.

First run

Quick Start

  1. Open a website you want to preview.
  2. Click Scan to extract colours and visual roles.
  3. Choose a curated theme such as Editorial Ink or Product Clarity.
  4. Click Apply to transform the visible page.
  5. Use the before/after toggle to compare the original and transformed views.
  6. Review the coverage score and contrast impact before exporting.

Design controls

Theme Studio

Theme Studio is the control surface for changing colour roles, typography, border radius, and density. Prism maps source styles to target tokens by role, so a button background, page background, link colour, and error state can remain distinct even when the original site reused similar raw CSS values.

Transformation

Live Website Preview

Prism injects an override stylesheet into the active tab and tags mapped elements with semantic roles. The original site remains untouched. The preview can be toggled off at any time, and saved themes can be associated with a domain locally.

The first target is the visible viewport. Future releases will deepen coverage for dynamic applications, component libraries, and state-specific styles.

Regression checks

Accessibility Impact

Prism compares the original and transformed theme for visual accessibility changes. The deterministic engine checks contrast ratios, focus visibility, and newly introduced regressions. AI may later explain findings, but pass/fail decisions should remain deterministic.

  • Resolved contrast issues
  • Newly introduced contrast issues
  • Improved or regressed text pairs
  • Focus indicator visibility
  • Coverage limits that affect confidence

Handoff

Exporting Themes

Prism exports CSS variables and design-token JSON so designers and developers can move from exploration to implementation. Exports are intended to be reviewed, adapted, and committed by the owning development team. 

:root {
  --prism-page-background: #f8f7f3;
  --prism-primary-text: #171717;
  --prism-primary-action: #0f4c81;
}

Implementation

Developer Integration

The MVP does not modify source code or open pull requests. Developer integration starts with clean exports, stable token names, and clear role mappings. Later phases may add a CLI, SDK, Figma plugin, automated source-code integration, and pull request generation.

Compatibility

Known Limitations

Prism works with most conventional DOM and CSS-based websites. It should not claim universal compatibility.

  • Canvas and WebGL content
  • Text embedded inside images
  • Closed Shadow DOM
  • Cross-origin iframes
  • Highly dynamic applications
  • Sites that aggressively regenerate inline styles
